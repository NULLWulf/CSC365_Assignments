Calculation Process

  • 1) Pull in Business JSON DATA: First Bussiness JSON data is loaded into memory as a glob.
  • 2) Add Bussinesses to Map: Glob is then iterated through and Bussinesses that are Restraunts and have a Review Count of at least 100 are added to the map. (This value can be tweaked globally on the backend
  • 3) Iterate Through Review JSON data: Then the Review JSON file is iterated through line by line. Because this JSON file is so large it becomes neccesary to buffer it vs pull the entire text contents into memory as I did with the Business JSON. A globally adjustable limit is set to 25,000 reviews.
  • 4) Tokenize/Generate Raw Count:
    • For each review the Business map is iterated through to attempt to find a match. When the Business ID of a Review and Business match, then Review's text is then cleaned of stop words, split into an array by spaces, and a raw term frequency count is appended to the respective business RawTermFrequency map. This is a map that stores the term frequency count for a term across all of it's respective reviews. Since I am only getting term frequency counts for businesses, the review data itself is volatile and does not get directly associated with the Business object.
